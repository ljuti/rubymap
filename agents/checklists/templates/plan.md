# Plan Checklist

## Scope
- [ ] All requirements from the spec are covered
- [ ] Implementation items are ordered by dependency
- [ ] Each item has a clear anchor (file/symbol)

## Acceptance
- [ ] Each item has behavioral acceptance criteria
- [ ] Acceptance criteria are testable
- [ ] Items are sized for single-pass implementation

## Completeness
- [ ] Edge cases have explicit items
- [ ] Error handling has explicit items
- [ ] No gaps between plan items
